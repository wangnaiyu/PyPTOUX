# Preview Gate

当现有 PTO system 不能满足新的共享视觉或交互模式时，必须先走 preview gate。

## Required Preview

preview 至少包含：

- 最接近的现有系统 pattern
- 拟新增 pattern
- normal / hover / active / selected 等状态覆盖
- token 使用说明
- 为什么当前系统不足

## Approval Rule

- 未明确批准：只保留 preview，不进入业务模块，也不投影到 `07-designsystem/`
- 明确批准：先吸收到 `.agents/skills/pto-design-system/`，再投影到 `07-designsystem/`，最后由业务模块消费

## Explicit Approval Examples

- “可以，就用这个”
- “approve”
- “按这个迁移”

## Not Approval

- “先看看”
- “接近了”
- “再调一调”
