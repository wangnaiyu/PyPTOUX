BYO-AI Domain Agent Workspace

PyPTOUX 如何把通用 AI 组织成特定领域 Agent

当前工程本身不内置模型，也不要求用户启动独立 agent 服务。它把 `AGENTS.md`、项目级 skills、知识库、PTO 设计系统、数据等级规则和协作约定打包成一个工作空间。用户用自己的 Codex 或 Claude 打开仓库后，AI 会按仓库规则完成检索、设计、原型和落盘。

查看作业流程 分享定位
不是内置模型 AI 能力来自用户自己的 Codex、Claude 或同类工具。
是领域工作空间 仓库提供任务路由、知识、约束、素材和验证规则。
接近 Agent Kit 打开项目即可获得 PyPTO/CANN 领域化的 AI 协作体验。

运行逻辑

workspace loop
01 用户提出任务 问业务知识、做 UX 设计、填 demo 素材、改前端原型或整理 GitHub。
02 Agent 读取项目入口 `AGENTS.md` 定义技能索引、协作边界、写作语言和数据治理。
03 按 intent 选择 skill 内容路由、知识检索、demo 数据填充、PTO 设计系统、Git/GitHub 各司其职。
04 检索证据并执行 优先用仓库知识库、本地 mirror、样例数据、官方文档和设计系统。
05 验证、落盘、交付 输出结果，同时维护 notes、prompts、index 或原型文件。
Positioning

它的准确定位

PyPTOUX 当前最像一个可分享的“领域 Agent 工作空间”。它把领域上下文、任务规程和产出治理放进仓库，让用户自己的 AI 在这个边界内工作。

用户自带 AI

模型、推理、工具调用能力来自用户自己的 Codex、Claude、Cursor 或 OpenCode。PyPTOUX 不试图替代这些通用 AI。

Workflow

一次任务的标准作业流程

当前 workflow 不是一个后台服务，而是由 Codex 或 Claude 在 workspace 中执行的 agent loop。规则写在仓库里，执行主体是用户打开项目后的 AI。

1

接收任务

用户直接问问题或要求产出，例如“解释 PyPTO 某机制”“做一个 PTO 风格原型”。

2

判断 intent

先判断是知识检索、内容路由、demo 数据、设计系统、原型实现还是 GitHub 操作。

3

加载 skill

只读匹配任务的 `SKILL.md` 和必要 reference，避免把所有规则混在一起。

4

检索证据

需要事实时回到本地 mirror、官方文档、样例数据或知识库，保留上游 literal。

5

执行产出

写文档、生成 HTML、整理原型素材、更新索引、运行验证或准备 Git 操作。

6

回写状态

把复合 prompt、update notes、sample-data、review 或 index 同步到对应目录。

Skill 的价值在于把“通用 AI 应该怎么做”变成稳定的项目协议。每个 skill 只覆盖一类任务，让 agent 可以先分类、再行动。

Dual Agent

Codex 与 Claude 的协作分工

PyPTOUX 当前默认是双 agent 协作模型。Codex 更像结构、事实和工程 owner；Claude 更像视觉、交互和前端原型 owner。

Codex 主责

规划 拆任务、定目录、整理方案和后续状态。
事实 查 PyPTO、CANN、样例数据和 source registry，验证字段、路径和 claim。
结构 维护知识库、索引、prompts、notes、sample-data 和 Git/GitHub 流程。
工程 处理构建、脚本、后端、测试、提交和 PR。

Claude 主责

原型 快速生成 HTML/JSX 前端验证稿和交互体验。
视觉 基于 PTO 设计系统做页面、工作台、graph、timeline 和 inspector 表达。
走查 记录原型验证、待办、体验问题和交互建议。
边界 涉及事实或 L2 推导时标记待 Codex review，不直接覆盖事实 owner。
Persistence

为什么它不是普通聊天

普通聊天的上下文会消失。PyPTOUX 的核心是把有价值的上下文变成可版本化文件，让后续 agent 能接着工作。

任务产物会进入稳定目录

  • `02-knowledge/` 保存业务知识和 shared framework。
  • `04-uxdesign/` 保存正式设计输出和决策记录。
  • `05-prototypes/` 保存原型、实验、样例数据和验证 notes。
  • `09-docs/03-indexes/` 让后续检索保持稳定。

过程信息也会被保留

  • `prompts/` 保存可再次执行的复合 prompt，不是原始聊天逐字稿。
  • `notes/update-YYYY-MM-DD.md` 记录本轮更新、验证结果和待确认问题。
  • `notes/story-YYYY-MM-DD.md` 记录用户场景、叙事脚本和 demo story。
  • `notes/sample-data.md` 标注数据等级、来源和生成规则。
Shareable Direction

作为可分享 Agent 的定位

最好的分享方式不是说“这是一个独立 AI”，而是说“这是一个自带 AI 的领域 Agent Kit”。用户下载后，把仓库设为工作空间，就能让自己的 AI 按 PyPTOUX 的规则工作。

PyPTOUX packages domain knowledge, skills, source-routing rules, design-system constraints and workflow memory, so a user's own AI can operate as a PyPTO/CANN domain assistant.
