# UX Design

这里用于存放需求分析、信息架构、交互方案、视觉方向、评审稿等正式设计输出。

- `shared/`：跨课题共用的设计原则与模式
- 具体课题目录：面向单一专题的设计产出

约定：

- 正式设计输出可使用 `.md` 或 `.html` 等便于阅读的格式；格式不单独决定目录归属。
- `notes/` 不是 Claude 原型验证、待办、备忘的默认位置；这些内容默认放到 `05-prototypes/<topic>/notes/`。
- 如确需在本目录保留 review notes，必须由 Codex 接收后落盘，并在文件头标明 `owner`、`status`、`decision`。
