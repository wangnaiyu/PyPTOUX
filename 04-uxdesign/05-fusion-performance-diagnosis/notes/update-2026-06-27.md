## 决策

- 将产品方向定义为 `Fusion Performance Diagnosis`，而不是 `Memory Viewer` 或通用 `Trace Viewer`。
- P0 聚焦慢 task 自动归因、分段泳道和 Memory Pressure，优先解决“看见慢但不知道为什么”的核心痛点。
- P1 承接 Wait Chain 与 Pipeline Balance，避免内存搬运、等待和流水利用率问题互相伪装。
- P2 承接源码 / 计算图 / `program.json` 联动与 Before / After 自动对比，完成行动和验证闭环。
- Demo 数据选择 A5 / Ascend 950 单 die 口径，而不是 910B / A2 口径；使用 `CORE_PER_DIE = 18`、`AIV_RATIO = 2`、`AIC_AIV_PER_DIE = 54` 生成完整泳道。
- Demo 数据等级标为 `L2 / schema-generated / share-safe`；字段形态、trace key 和泳道口径来自 source，具体耗时、PMU counter、memory bytes 和 contributors 为按 issue 场景生成的示意数据。
