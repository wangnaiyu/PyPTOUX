PyPTO Design System

基于 Shadcn 实现的 PTO 3.0 设计风格组件库，配合 design.md 和 skill 使用。本页面展示基础组件、tokens 和 patterns 入口。

Agent Reference
Token Stack (load order)
tokens/foundation.cssPrimitives — raw color palette, spacing scale, radius, font families
tokens/semantic.cssSemantic aliases — --background, --foreground, --surface-1..4, --border-*, --state-*
tokens/components.cssComponent tokens — --button-*, --card-*, --panel-*, --tag-*, typography composites
css/style.cssClass rules — .btn, .btn-solid, .btn-ghost, .btn-icon, .is-selected, layout primitives
patterns/patterns.jsonPattern registry — graph/timeline primitives that must be reused from their source files instead of copied into preview CSS.
tokens.json — 生成产物说明
文件路径：tokens/tokens.json　　生成脚本：tokens/generate_tokens.py
$meta生成时间、生成器、source-of-truth 说明。用于明确该文件不可手改。
source.foundation来自 tokens/foundation.css 的原始 token，按 base / light 分组。
source.semantic来自 tokens/semantic.css 的语义 token，按 dark / light 分组。
source.components来自 tokens/components.css 的组件 token，按 base / light 分组。
themes.dark / themes.light由三份 CSS token 源合并后的最终主题变量快照。键为 CSS 变量名（不含 --）。
使用原则三份 CSS token 源才是单一来源。修改样式 → 改 CSS token 源 → 运行生成器；不要直接编辑 tokens.json 或 tokens.js。

Canonical Button Classes
.btnSoft / secondary. Surface fill + border + secondary text. Default for most actions.
.btn.btn-solidPrimary CTA. var(--foreground) fill + var(--background) text. High-contrast. One per context.
.btn.btn-ghostUtility / icon-adjacent. No bg, muted text, hover reveals bg. Toolbar helpers, back links.
.btn.btn-iconSquare icon-only size modifier. Pair with any variant: .btn.btn-icon, .btn.btn-icon.btn-ghost.
.btn[disabled]Disabled state. Opacity 0.42, pointer-events none. No extra class needed.
.btn.is-selectedSelected / active toggle state. Same fill as .btn-solid. Overrides all variant backgrounds.
Component Usage Notes
Button weight.btn-solid is for the only action that commits or advances the workflow: Run, Generate, Apply, Execute, Start. Open/import/browse/load actions stay .btn.
Ghost action.btn-ghost is for cancel, close, more, toolbar helpers, and icon-only utility actions.
Outline toggleSingle binary toggles use .toggle-outline with aria-pressed. Do not fake toggle state with generic buttons.
Toggle groupMutually exclusive options use grouped outline toggles. Keep size smaller than primary actions.
Chip filterFilter chips stay low-emphasis. Selected chips use a muted selected surface, not a solid CTA fill.
Status badgeSemantic badges keep the neutral badge structure and only change text/border color.
Action listAction lists live inside card content. One row may be selected; do not promote each row to a separate component category.
Foundation Rules
Semantic mappingNeutral ramp values are primitive colors. Map them through theme-level surface, foreground, border, and state tokens before use.
Highlight stepsLight themes use low steps for backgrounds; dark themes use high steps for backgrounds. Middle steps stay for foregrounds or data-viz base hues.
State overlayHover, pressed, muted, selected, and focus states come from semantic state tokens. Do not hard-code per-component alpha overlays.
Palette policyShared tokens only hold reusable palettes. Module-local alpha fills stay local unless promoted through a documented token pass.
State Modifier Rules
.is-selectedUniversal selected state. Applies to .btn of any variant. Produces var(--foreground) fill + var(--background) text. Used by toggle groups, chip filters, toolbar active states.
DeprecatedDo NOT use: .btn-flat, .btn-md-primary, .btn-md-text, .sw-toggle-active with blue fill, --button-flat-primary-* tokens. All removed or neutralised.
Pattern Reuse Rules
Stable entryEvery pattern must expose a stable consumer entry. Static visual primitives use documented CSS classes; DOM/SVG/canvas generation, drag behavior, zoom, routing, persistence, or synchronized state must go through a window.Pto*Pattern or window.Pto* helper. Business pages must not copy generated DOM or restyle internal pattern classes.
swimlane-task-barUse patterns/swimlane-task/pattern.js and call window.PtoSwimlaneTaskPattern.drawTaskBar, createTaskColormap, and initHoverTooltip. The canonical task bar is canvas-rendered, sourced from pypto-swimlane-perf-tool/js/swimlane.js, and the hover tip mirrors the PMU swimlane .sl-tooltip interaction. Do not rebuild it with DOM/CSS or rewrite segment math, colormap selection, or hover tip behavior locally.
memory-architecture-layoutUse patterns/memory-architecture/pattern.js. Call window.PtoMemoryArchitecturePattern.renderArchitecture to build the stage, createRouteOverlay for MTE routes, attachHoverInteractions for hardware tips, and attachPathFocusInteractions or setPathFocus / clearPathFocus for ascend-hardware-map-v3-style flow-step hover. Extend preset data for 910B/950; do not copy raw mem_viewer HTML, redraw route geometry locally, reimplement is-path-focused route glow, or change route stroke-width for hover/playback focus.
aic-core-objectUse patterns/aic-core-object/pattern.js and call window.PtoAicCorePattern.render. The shell is config-driven from the design draft; extend the preset data for 950B instead of cloning the generated DOM or restyling the chrome locally.
aiv-core-objectUse patterns/aiv-core-object/pattern.js and call window.PtoAivCorePattern.render. The shell is config-driven from the approved AIV draft; extend the preset data for later chip variants instead of copying the generated DOM or changing card geometry locally.
workbench-shellResize kernel only. Use patterns/workbench-shell/pattern.css and patterns/workbench-shell/pattern.js, then call window.PtoWorkbenchShell.initResizablePanes, createSplitGutter, or initNestedResizablePanes for horizontal / vertical / nested splits. Do not put page chrome, pane fills, topbars, titles, canvas controls, or IDE typography into this pattern.
ide-frameIDE framework shell. Use patterns/ide-frame/pattern.css, patterns/ide-frame/pattern.js, patterns/floating-playback-control/pattern.css, patterns/floating-playback-control/pattern.js, and optional patterns/ide-frame/vscode.css. Set data-host="standalone" or data-host="vscode-webview". It owns IDE chrome, pane headers, pane-local preview tabs, generic slots, floating playback mounting, and split initialization. It must not ship business sample data, must not recreate playback chrome, and must not override .pto-workbench-shell__* internals.
floating-playback-controlUse patterns/floating-playback-control/pattern.css and patterns/floating-playback-control/pattern.js. Pages should use .pto-floating-playback* classes or call window.PtoFloatingPlaybackControl.createControl, then call init and initScrubberHover. Business pages own playback data and button handlers, but must not copy floating-shell chrome, collapse synchronization, or scrubber hover behavior locally.
model-graphvizUse patterns/model-graphviz/pattern.html for parity preview; it iframes the real DeepSeek V3.2 Graphviz page directly. For new lightweight examples, use patterns/model-graphviz/pattern.js and call window.PtoModelGraphvizPattern.render. The contract preserves the DeepSeek V3.2 rules: default depth-2 folding, full-pill module/op nodes, rounded-rect tensor nodes for Input/Output/MTP/Constant/Parameter/Tensor, graph stage, graph edges, safe colormap normalized to s=0.82/l=0.40, 50% light-mode node-fill opacity, 10% white expanded-cluster fill for every container (including repeated layer templates), --radius-xl (16px) cluster corners, vertically centered node label+type stacks, strict semantic module hierarchy, parent-to-child fill inheritance for op/module nodes, folded representative edge dedupe, stable expand/collapse viewport anchoring, top-right corner fold controls, right-inset expand controls, no graph shadows, and no-border clicked-operator preview cards. New model graph pages additionally derive cluster boxes from their visible-child bounding boxes (so collapse reflows and shrinks the parent), lay the decoder chain out as a single top-to-bottom column (Gate/Up the only parallel pair), use transform-based cursor-anchored pan/zoom, show a single white selection stroke with no browser focus ring, and place the repeat-count badge at the cluster top-left under the title; qwen7b_modelviz.html is the reference implementation. Node fill colors are the only data-viz color exception; all other UI styling must remain token-derived. Do not render tensor nodes as ellipses or full pills, drift fold icons away from the cluster top-right corner anchor, use colored expanded-cluster fills, add graph shadows or operator-card borders, hard-code non-node UI styles, recenter/refit the whole graph after expand/collapse, or duplicate an expanded parent as a standalone node.
model-training-graphvizUse patterns/model-training-graphviz/pattern.js after loading model-graphviz. Call window.PtoModelTrainingGraphvizPattern.render with graph, edge tags, and trainingEvidence; use controller.setPhase({ nodeId, relatedNodeIds }) for training-step linkage. It owns hover evidence panels, edge type tags, selected/related emphasis, drag pan, and Command-wheel zoom. Do not iframe a second page shell, recenter the graph when phases advance, dim non-selected operators, create native title tooltips, draw source-file evidence columns inside the graph, or use saturated light-mode selection borders.
Preview rulePreview pages may wrap a pattern for layout, but must not redefine the pattern's internal class styles.
